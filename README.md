## 📌 Overview

This project builds a **supervised machine learning pipeline** to predict median house prices across California districts. It uses the well-known **California Housing Dataset** (built into scikit-learn), which contains census-derived features about neighborhoods — things like median income, average number of rooms, and population density — mapped to median house prices.

The project is intentionally structured as **four clean, single-responsibility Python modules**, each doing exactly one job. This makes the code easy to follow, easy to modify, and easy to extend.

## 📦 The Dataset — California Housing

The dataset is fetched directly from scikit-learn via `fetch_california_housing()` — no manual download needed.

It contains **20,640 samples**, each representing a California census block, with the following 8 features:

| Feature | Description |
|---|---|
| `MedInc` | Median income of households in the block (in tens of thousands of USD) |
| `HouseAge` | Median age of houses in the block |
| `AveRooms` | Average number of rooms per household |
| `AveBedrms` | Average number of bedrooms per household |
| `Population` | Total population of the block |
| `AveOccup` | Average number of household members |
| `Latitude` | Geographic latitude of the block |
| `Longitude` | Geographic longitude of the block |

**Target variable:** `PRICE` — the median house value for the block, in hundreds of thousands of dollars.

## ⚙️ How the Pipeline Works

The project is split into four scripts. Here's exactly what each one does:

---

### 1. `data_loader.py` — Data Ingestion & Splitting

This is the foundation everything else depends on.
```python
from sklearn.datasets import fetch_california_housing
from sklearn.model_selection import train_test_split
```

**What it does:**
- Fetches the California Housing dataset from scikit-learn
- Converts it into a Pandas DataFrame with named columns
- Adds a `PRICE` column as the target variable
- Splits into **training (80%)** and **testing (20%)** sets with `random_state=42` for reproducibility
- Returns `X_train, X_test, y_train, y_test, df` — the full toolkit every other module needs

**Why this matters:** By centralizing all data logic in one function, every other script calls `load_and_split_data()` and gets the exact same split. There's no risk of data leakage or inconsistency between training and testing.

---

### 2. `explore.py` — Exploratory Data Analysis (EDA)

Before training any model, it's essential to *understand* the data. This script does that.
```python
from data_loader import load_and_split_data
import matplotlib.pyplot as plt
import seaborn as sns
```

**What it does:**

- **Summary Statistics** — `df.describe()` prints count, mean, std, min, max for every feature. This immediately surfaces things like skewed distributions or unexpectedly large ranges.

- **Null Check** — `df.isnull().sum()` confirms whether any values are missing. (California Housing is clean, but this is good practice on any dataset.)

- **Correlation Heatmap** — A `seaborn` heatmap of the full correlation matrix, annotated with values, using a `coolwarm` color scale. This visually shows which features are most related to `PRICE` and to each other. Strong correlations (close to +1 or -1) stand out immediately.

- **Scatter Plot: `MedInc` vs `PRICE`** — Median income is the single strongest predictor of house price in this dataset. This scatter plot makes that relationship visually obvious — as income rises, price rises.

- **Histogram of `PRICE`** — A distribution plot with KDE (kernel density estimate) overlay showing how house prices are spread. This reveals if the target is normally distributed or skewed (it has a hard cap at $5.00, which shows up as a spike).

---

### 3. `train.py` — Model Training & Serialization

This is where the model is built.
```python
from sklearn.linear_model import LinearRegression
import joblib
```

**What it does:**
- Calls `load_and_split_data()` to get the training split
- Instantiates a `LinearRegression` model and fits it on `X_train` and `y_train`
- Prints a **coefficients table** — every feature alongside its learned weight, so you can see exactly how much each variable influences the predicted price
- Prints the **intercept** — the baseline price before any features are considered
- Saves the trained model to `linear_model.pkl` using `joblib`

**Sample output from `train.py`:**
```
        Feature    Weight
0        MedInc   0.4487
1      HouseAge   0.0097
2      AveRooms  -0.1073
3     AveBedrms   0.6451
4    Population  -0.0000
5     AveOccup  -0.0038
6      Latitude  -0.4196
7     Longitude  -0.4330

Intercept: -36.941...
Model saved as linear_model.pkl
```

Reading this tells a real story: `MedInc` has a strong positive weight (higher income → higher price), while `Latitude` and `Longitude` have negative weights (price decreases as you move north/east within California's geographic range).

**Why `joblib` instead of `pickle`?** `joblib` is the recommended way to serialize scikit-learn models — it handles large NumPy arrays more efficiently than standard `pickle`.

---

### 4. `test.py` — Model Evaluation & Visualization

This script answers the key question: *how good is the model?*
```python
from sklearn.metrics import mean_absolute_error, mean_squared_error, r2_score
import joblib
```

**What it does:**
- Loads the saved `linear_model.pkl` from disk
- Runs predictions on `X_test` (data the model has never seen)
- Computes three evaluation metrics:

| Metric | What it Tells You |
|---|---|
| **MAE** (Mean Absolute Error) | On average, how many $100k units off is the prediction? |
| **MSE** (Mean Squared Error) | Same as MAE but large errors are penalized more heavily (squared) |
| **R² Score** | What % of the variance in price does the model explain? (1.0 = perfect) |

- Plots **Actual vs Predicted Prices** as a scatter plot — if the model were perfect, all points would lie on a diagonal line. How tightly clustered the points are around that diagonal tells you how well the model generalizes.

## 🧠 Why Linear Regression?

Linear Regression fits a hyperplane through the feature space that minimizes the sum of squared errors between predicted and actual prices. It's the right starting point here because:

- **Interpretability** — The coefficient table from `train.py` gives a clear, human-readable explanation of what drives price up or down.
- **Simplicity** — With 8 numerical features and a continuous target, there's no need to start with a complex model.
- **Baseline value** — Any future model (Random Forest, XGBoost, Neural Network) should be benchmarked against this Linear Regression baseline. If a complex model doesn't significantly outperform it, the added complexity isn't justified.
